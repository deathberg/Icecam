# AGENTS.md

## Cursor Cloud specific instructions

IceCam is a single-module **Android application** (Gradle, AGP `8.9.0`, `compileSdk 35`,
`minSdk 23`). It is not a server/web app: the deliverable is an installable debug APK,
which is exactly what CI builds (`.github/workflows/android.yml` runs `gradle assembleDebug`).

### Toolchain (baked into the VM snapshot)
- **JDK 17** at `/usr/lib/jvm/java-17-openjdk-amd64` (CI uses Java 17; the default system Java is 21, which AGP 8.9 does not target).
- **Android SDK** at `~/android-sdk` (`platform-tools`, `platforms;android-35`, `build-tools;35.0.0`).
- **Gradle 8.11.1** at `/opt/gradle/gradle-8.11.1` symlinked to `/usr/local/bin/gradle`.
- `~/.bashrc` exports `JAVA_HOME`, `ANDROID_HOME`, `ANDROID_SDK_ROOT` and prepends them to `PATH`.
  Non-login shells may not source it; run `source ~/.bashrc` first, or rely on `local.properties`
  (`sdk.dir=~/android-sdk`, regenerated by the update script) so Gradle can locate the SDK.

### Build / lint / run
- Build (primary dev workflow): `gradle --no-daemon assembleDebug` → `app/build/outputs/apk/debug/app-debug.apk`.
- Lint: `gradle --no-daemon lintDebug`. NOTE: this currently **fails** on 4 pre-existing
  `NewApi` errors in `app/src/main/java/dev/icecam/app/runtime/FlightRecorder.java` (and others).
  This is a code issue, not an environment issue; do not "fix" it as part of env setup.
- **No automated unit/instrumentation tests exist** (`src/test`, `src/androidTest` are absent).
- There is **no Gradle wrapper** (`gradlew`); use the system `gradle`.

### Running the app (important caveat)
The app cannot be meaningfully run inside this VM: there is no `/dev/kvm` / hardware
virtualization (so no usable emulator), and the app is designed for a **rooted Xiaomi device
with the LSPosed framework** (it deploys native binaries via `su` and registers a Binder
service `com.xiaomi.vlive.IMyBinderService`). Validate changes by building/inspecting the APK
(`aapt dump badging`, `unzip -l`) rather than launching it.
